# ClipAnchor

[English](README.md) | 简体中文

ClipAnchor 是一款基于 **Tauri + React** 的桌面剪贴板辅助工具，重点提供剪贴板内容预览、悬浮固定、历史记录、收藏、隐私过滤、快捷键和扩展语言包等能力。应用以本地运行和本地数据存储为主，适合在日常办公、资料整理、开发和多窗口工作场景中使用。

> 本项目仍在持续迭代。下载、安装或升级前，建议先阅读发行说明并备份重要数据。

## 功能概览

- 支持文本、图片、文件和混合剪贴板内容。
- 复制后显示轻量预览弹窗，可固定、取消固定或手动关闭。
- 提供剪贴板历史、搜索、收藏、删除和数据清理。
- 支持文本记录编辑以及历史记录导入、导出。
- 支持隐私模式、内容类型过滤和自动销毁时间设置。
- 支持全局快捷键、深色/浅色/跟随系统主题及界面缩放。
- 支持开机启动、托盘运行、窗口位置记忆和更新检查。
- 内置简体中文与英文，并支持本地扩展语言包。
- 扩展语言包支持增量更新，避免对未变化内容重复调用翻译接口。

## 下载与安装

### 使用发行版

1. 打开本仓库的 **Releases** 页面。
2. 下载适合当前操作系统和 CPU 架构的安装包或便携包。
3. 安装或解压后启动 ClipAnchor。
4. 首次启动后，可在“设置”中调整快捷键、隐私过滤、主题、语言和自动销毁时间。

发行页面未提供某个平台的安装包时，表示该版本尚未为该平台构建，请使用源码构建。

### 便携版数据目录

便携版通常将运行数据保存在程序目录下的：

```text
data/
├── locales/     # 用户扩展语言包
├── logs/        # 日志
└── ...          # 设置、历史记录和其他运行数据
```

不要将 `data/` 目录提交到公开仓库，因为其中可能包含个人设置、日志或剪贴板数据。

## 从源码运行

### 环境要求

- Node.js 与 npm
- Rust 工具链
- Tauri 所需的系统编译环境
- Git

不同操作系统所需的系统依赖不同，请先完成 Tauri 桌面应用的开发环境配置。

### 安装依赖

```bash
npm install
```

### 开发模式

```bash
npm run tauri:dev
```

### 构建发行包

```bash
npm run tauri:build
```

如脚本名称发生变化，请以当前 `package.json` 中的 `scripts` 为准。

## 基本使用

1. 启动 ClipAnchor。
2. 在设置中启用“剪贴板固定服务”和“历史记录服务”。
3. 复制文本、图片或文件。
4. 在预览弹窗中固定内容，或等待未固定弹窗自动关闭。
5. 在主窗口中搜索、收藏、复制、编辑或删除历史记录。
6. 使用设置页调整快捷键、主题、语言、缩放、过滤规则和数据保留策略。

## 扩展语言包

### 内置语言与扩展语言

内置语言：

- `en`：English
- `zh-CN` / `zh-Hans`：简体中文

其他语言通过 JSON 扩展语言包加载。语言包文件必须放在软件运行数据目录：

```text
data/locales/
```

复制完成后，**完全退出并重新启动 ClipAnchor**。重启后，该语言会出现在“设置 → 外观 → 扩展语言”中。

### 自行翻译时参考哪些文件

- 英文原始文案与键名来源：[`src/locales/en.js`](src/locales/en.js)
- JSON 格式模板：[`docs/i18n/language-pack.template.json`](docs/i18n/language-pack.template.json)
- 详细制作说明：[`docs/i18n/README.md`](docs/i18n/README.md)

翻译时必须以 `src/locales/en.js` 中 `messages` 对象的键名为准。**只翻译值，不要翻译、删除或随意修改键名。**

### 文件名和语言代号

语言包文件名必须使用 **IETF BCP 47** 风格的语言标签，并以 `.json` 结尾。

推荐写法：

| 语言 | 文件名 |
|---|---|
| 日语 | `ja.json` |
| 法语 | `fr.json` |
| 德语 | `de.json` |
| 西班牙语 | `es.json` |
| 巴西葡萄牙语 | `pt-BR.json` |
| 繁体中文（台湾） | `zh-TW.json` |
| 塞尔维亚语（拉丁字母） | `sr-Latn.json` |

命名规则：

- 使用连字符 `-`，不要使用下划线 `_`。
- 主语言通常小写，例如 `fr`、`ja`。
- 书写系统首字母大写，例如 `Latn`、`Hant`。
- 国家或地区通常大写，例如 `BR`、`TW`。
- 文件名不应使用空格或自定义描述，例如不要使用 `French.json`、`繁體中文.json`。
- JSON 中的 `code` 应与文件名去掉 `.json` 后一致。
- `auto`、`en`、`en-*`、`zh`、`zh-CN` 和 `zh-Hans*` 属于自动或内置语言范围，不应作为普通扩展语言包名称。

### 最小可用格式

```json
{
  "format": "clipanchor-language-pack",
  "code": "fr",
  "label": "French",
  "native_name": "Français",
  "source": "manual",
  "source_locale": "en",
  "messages": {
    "settings": "Paramètres",
    "cancel": "Annuler",
    "ok": "OK"
  },
  "message_status": {}
}
```

实际语言包应包含 `src/locales/en.js` 中的全部界面键。上面的示例只用于说明结构，不能作为完整语言包直接发布。

格式要求：

- 文件编码必须为 UTF-8。
- 必须是合法 JSON，不能写注释，最后一项后不能有多余逗号。
- `messages` 必须是对象，键和值都应为字符串。
- 保留原文中的占位符，例如 `{language}`、`{count}`、`{error}`、`{days}`。
- 保留换行和转义含义；JSON 中换行写成 `\n`，双引号写成 `\"`。
- 不要把 API Key、剪贴板内容或其他私密数据写入语言包。
- `message_status` 可以先留空；软件首次扫描时会为旧格式或手工语言包建立状态基线。

### 如何判断语言项是否需要更新

ClipAnchor 以当前英文界面文案为基准，对每个语言键进行检查：

| 状态 | 判断方式 | 处理 |
|---|---|---|
| 缺失项 | 英文源文件存在该键，但语言包 `messages` 中不存在 | 标记为需要新增翻译 |
| 原文已变化 | 当前英文文本的 `source_hash` 与语言包记录不同 | 标记为需要更新 |
| 已删除项 | 语言包存在该键，但当前英文源文件已不存在 | 增量更新时本地删除，不调用翻译接口 |
| 人工修改 | 当前译文的 `translation_hash` 与记录不同 | 将 `modified` 标记为 `true`，避免自动更新覆盖人工译文 |
| 文件错误 | JSON 无法解析、结构无效或没有可用 `messages` | 标记为损坏，需修复或重新生成 |

`source_hash` 和 `translation_hash` 是用于比较文本是否变化的轻量指纹，不是安全加密哈希。

增量更新原则：

1. 只处理缺失项和英文原文发生变化的项。
2. 未变化译文直接复用。
3. 已从软件删除的键只在本地清理。
4. 检测到人工修改的译文时优先保留人工内容。
5. 人工修改项对应的英文原文后来发生变化时，建议译者手动复核语义，避免旧译文与新原文不一致。

## 语言包更新与故障排查

- **“有可用更新”**：语言包仍可使用，但存在新增、变化或已删除的键，可执行增量更新。
- **“文件错误/损坏”**：文件不是合法 JSON、结构错误或没有有效翻译，修复前不应启用。
- 修改文件后没有显示：
  1. 确认文件位于 `data/locales/`；
  2. 确认扩展名是 `.json`；
  3. 确认文件名是有效语言标签；
  4. 用 JSON 校验工具检查格式；
  5. 完全退出托盘中的 ClipAnchor 后重新启动。
- 界面出现英文回退：通常表示某些键缺失，请与 `src/locales/en.js` 对照补全。
- 占位符显示异常：检查译文是否保留了原始 `{...}` 占位符。

## 隐私与数据安全

ClipAnchor 会处理用户复制的文本、图片和文件路径。请注意：

- 不要在公共设备上长期保存敏感剪贴板内容。
- 根据实际需要启用隐私过滤和自动清理。
- 发布日志或提交 Issue 前，检查其中是否包含文件路径、账号、令牌或剪贴板内容。
- 不要将 `data/`、`.env`、API Key、签名证书和本机构建凭据提交到 Git。
- 删除本地数据前先备份需要保留的记录。
