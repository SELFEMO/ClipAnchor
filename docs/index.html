Local first, with maintenance controls.

ClipAnchor is built as a desktop utility, not a cloud clipboard. Sensitive content filtering, pause controls, and local logs help keep behavior understandable when you need to troubleshoot.

Sensitive content filter

Light mode uses local rules to avoid storing likely passwords, keys, IDs, tokens, and similar text.

Rotated runtime logs

Logs record service, tray, popup, shortcut, data, and error events. Clipboard body text is not written to logs.

One running instance

Starting ClipAnchor again activates the existing main window instead of leaving another long-running process.

FAQ

Where is my data stored?

All runtime data is stored under the app-local data folder, including clipanchor.db, settings, resources, exports, and logs.

What happens at system startup?

ClipAnchor starts in Lite mode by default. The main window stays hidden while tray, monitoring, and storage services keep running.

Can I export history?

Yes. JSON export keeps full data, and CSV export includes record metadata such as type, paths, timestamps, size, hash, and favorite state.

How do language and color mode work?

The page follows your system language and color mode on first visit. Manual choices are remembered locally in your browser.
